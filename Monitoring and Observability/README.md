# 📈 Monitoring & Observability

If you can't measure it, you can't manage it. Learn how to use CloudWatch to track performance, health, and costs.

## 🗺️ Lab Roadmap

| Lab | Goal | Description |
| :--- | :--- | :--- |
| [**Lab 1: CloudWatch Alarms**](./Lab%201%20-%20CloudWatch%20Alarms/README.md) | **Alerting** | Set up CPU and Billing alarms with SNS notifications. |

---

## 🛠️ What You'll Learn
- The difference between **Metrics**, **Logs**, and **Alarms**.
- Monitoring EC2 health (CPU, Disk, Network).
- Setting up **Billing Alarms** to avoid "AWS Bill Shock."
- Connecting CloudWatch to **SNS** for email alerts.
